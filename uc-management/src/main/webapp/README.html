src/main/webapp中的内容

  • metronic - metronic原有的目录，未做任何改动，放在这里只是方便开放，在实际打包的时候会将其排除在外
  • resources - 保存的是图片、脚本和CSS，完全拷贝metronic中的相应目录（metronic/admin/templates/admin/assets）
    但是有些链接改成了正确的路径。
  • index.jsp - 该页面实现自动跳转，由com.runmit.uc.management.controller.IndexController.java决定
  • WEB-INF/inculde - 封装的页面组件，这些组件会在WEB-INF/views/frame.jsp中引用，包括：
    1. cover.jsp - 首页内容，实际项目中另外制作首页，不要使用这个说名
    2. global_head.jsp - 全局head，包括基本的css、javascript、自定义脚本
    3. header.jsp - 页面header
    4. footer.jsp - 页面footer
    5. sidebar.jsp - 页面左面的导航栏
    6. portlet_configuration.jsp - 未使用，是关于页面设置的
    7. quick_sidebar.jsp - 最右上角的图标点击后出现的弹窗
    8. search_box.jsp - 未使用，搜索栏
    9. sidebar_toggler.jsp - 未使用，隐藏页面左面的导航栏
    10. style_customizer.jsp - 页面右面齿轮图标触发的窗口，用来修改主题和显示设置
  • WEB-INF/views - 需要开放的页面，可以参考下面关于[页面的制作]部分

页面的制作

所有页面都是基于Metronic的页面，具体路径是metronic/admin/templates/admin/*.html
如果需要新的组件在Metronic中查找到相关页面，然后进行以下操作:
Tips:所有位置都以注释的形式说明，使用IDE的折叠代码功能可以很快定位和操作

  1. 从WEB-INF/views/example中拷贝已有的页面
  2. 将原始HTML中的PAGE LEVEL PLUGIN STYLES中的内容放入本页中对应的位置（文件开始部分），
    注意将路径中的
    ../../assets
    替换为
    ${ctx}/resources
  3. 将原始HTML中的PAGE LEVEL STYLES中的内容放入本页中对应的位置（紧跟上面的PLUGIN STYLES），
    注意将路径中的
    ../../assets
    替换为
    ${ctx}/resources
  4. 更改本页的PAGE HEADER中的页面名称、页面概要、导航栏
  5. 将原始HTML中的PAGE CONTENT中的内容放入本页中对应的位置（文件中间部分）
  6. 将原始HTML中的PAGE LEVEL PLUGINS中的内容放入本页中对应的位置（文件结尾部分），
    注意将路径中的
    ../../assets
    替换为
    ${ctx}/resources
  7. 将原始HTML中的PAGE LEVEL SCRIPTS中的内容放入本页中对应的位置（紧跟上面的PLUGINS），
    但是注意不要包含metronic.js、layout.js和quick-sidebar.js，它们已经在global_head.jsp中定义了，
    注意将路径中的
    ../../assets
    替换为
    ${ctx}/resources
  8. 将原始HTML中的jQuery(document).ready函数中的初始化命令放入本页的对应位置中，
    但是注意不要包含Metronic.init();Layout.init();QuickSidebar.init() ，它们也同样在global_head.jsp中定义了
  9. 在java代码中开发对应的controller，定义页面路径。然后修改WEB-INF/include/sidebar.jsp中的导航项的引用地址
